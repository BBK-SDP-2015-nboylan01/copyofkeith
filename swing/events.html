Event Delegation Model

§ A listener object is an instance of a class that implements a special interface called a "listener interface"

§ An event source is an object that can register listener objects and send them event objects.

§ An eligible listener registers itself with an event source by calling an "add listener" method in the event source and passing a reference to itself ("callback");  multiple listeners can register with the same source (multicasting)
 

eventSourceObject.addEventListener(eventListenerObject);


§ The event source sends out event objects to all registered listeners when that event occurs.

§ The listener objects can use the information in the event object to determine their reaction to the event.
 

How to implement an Event Handler:

  1. In the declaration for the event handler class, add code that specifies that the class either implements a listener interface or extends a class that implements a listener interface

    e.g. public class MyClass implements ActionListener{ ... }

  2. Add code that registers an instance of the event handler class as a listener upon one or more components.

    e.g. someComponent.addActionListener(instanceOfMyClass);

  3. Add code that implements the methods in the listener interface.

    e.g. Add the following to class MyClass:

    public void actionPerformed(ActionEvent ae)
        {
        // code that reacts to the event
        }
